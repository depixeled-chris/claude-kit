(bug) cap.mjs ROUTES BY CWD-WALK — misroutes cross-project captures. findAiDir(process.cwd()) walks UP from the shell cwd to the nearest .ai/config.yml, so when the shell sits in the claude-kit repo (running q/survey/etc.) a `cap` of a HOD decision lands in the KIT inbox. Result 2026-06-06: ~19 HOD/engine captures misfiled into claude-kit's inbox (GOING RAPIER PHYSICS, one-rust-core, roads, terrain, R052/R053 bugs…). The agent's cwd is NOT a reliable signal of the target project. Fix: route by the ACTIVE/target project — accept `--project <name>` (resolve via the registry), and/or default to the registry's active project, and WARN when capturing into a project other than the one whose work is in flight. Consider a triage-time guard that flags an inbox item whose body says "Project: X" but sits in Y's store. Same class of failure as the retrieval gate (KIT-T050): the tool should dictate correctness, not rely on the agent getting cwd right.
