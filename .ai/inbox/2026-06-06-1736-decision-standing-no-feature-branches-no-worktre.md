(decision/standing) NO FEATURE BRANCHES / NO WORKTREE-ISOLATED SUBAGENTS WITHOUT EXPLICIT AGREEMENT. Default: work on the ONE agreed working branch; commit + push directly to it. Do NOT spin up feature branches or isolation:'worktree' subagents unless the maintainer and agent both agree first. NEVER run multiple worktree subagents on separate branches on the same filesystem — it caused (this session): the primary checkout silently slipping to a stale branch, agents cutting from STALE bases, and messy cross-branch merges. Subagents that edit code should work on the SAME branch the orchestrator is on (or be read-only/advisory), unless a genuine parallel-conflict case is explicitly agreed.
